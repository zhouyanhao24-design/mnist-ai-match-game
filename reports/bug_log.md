# Bug 记录与修复过程

## Bug 1：输入通道不匹配

现象：训练时报错，卷积层输入通道数不一致。

原因：MNIST 是灰度图，输入通道数为 1。

修复：在 `models.py` 中把 ResNet 和 VGG 的第一层卷积输入通道设置为 1。

## Bug 2：三个相同数字没有自动消除

现象：点击三个相同预测数字后，卡牌仍然留在卡槽中。

定位：检查 `remove_triples()` 函数，发现需要按预测标签分组统计。

修复：使用字典统计每个预测数字对应的卡牌，达到 3 个时标记为 removed 并从卡槽中移除。

## Bug 3：没有训练模型时游戏无法演示

现象：没有模型文件时运行游戏失败。

修复：如果 checkpoint 不存在，游戏暂时使用真实标签演示；训练模型后再使用模型预测。

## Bug 4：错误率过高导致游戏难度过高

现象：`error-rate` 较大时，卡槽更容易堆满。

解释：错误预测会使本来可以匹配的数字被分散为不同类别，因此失败概率增加。
